# AGENTS.md — Institutionsguide.dk

## Build & Run

```bash
npm install
npm run dev              # Dev server (localhost:5173)
npm run build            # tsc -b && vite build + prerender (1939 pages)
npm run preview          # Preview production build
npm run test:run         # Vitest (non-watch)
npm run lint             # ESLint
```

## Backpressure

1. `npm run build` — zero errors, zero warnings
2. `npx tsc -b` — zero type errors
3. `npm run test:run` — all tests pass
4. `grep -r "console.log" src/ --include="*.ts" --include="*.tsx" | grep -v node_modules | grep -v ".test."` — zero hits
5. Build output: no chunk over 250KB (except lazy-loaded vendor chunks)

## Stack

- **Framework**: React 19 + Vite 8 + TypeScript 5.9
- **Styling**: Tailwind 4 + @tailwindcss/vite plugin. Fonts: Fraunces (display) + DM Sans (body).
- **Routing**: React Router v7, lazy-loaded via `lazyRetry()` wrapper in App.tsx. 40+ routes.
- **State**: DataContext (institutions data), LanguageContext, CompareContext, FamilyContext. localStorage for favorites + recently viewed.
- **Backend**: Supabase (auth, DB, edge functions). Edge functions: analyze-reviews, budget-ai, onboarding-ai, assessment, google-rating.
- **Maps**: Leaflet + react-leaflet + MarkerCluster
- **Charts**: Recharts 3
- **SEO**: prerender-pages.mjs generates 1939 static HTML pages at build. SEOHead component + JsonLd component. generate-sitemap.mjs.
- **i18n**: da.ts + en.ts (~1500 strings each). LanguageContext + useLanguage hook.
- **Error tracking**: Sentry
- **Deployment**: Vercel, auto-deploy from master branch.
- **Data pipeline**: 27+ scripts in scripts/ fetching from DST, STIL, Arbejdstilsynet, forældretilfredshed, Google Places.

## Architecture Patterns

- Path alias: `@/` maps to `src/`
- Gate model: `InstitutionGateModal` + `GatedSection` — email capture required for detailed data
- Scoring: `institutionScore.ts` computes composite quality score. `preferenceScore.ts` for personalized ranking.
- AI: `InstitutionChat.tsx` + `useInstitutionChat.ts` — Claude Haiku via Supabase edge function
- Data: Static JSON in `public/` + Supabase queries. `DataContext` loads and provides.
- Pre-rendering: `prerender-pages.mjs` generates static HTML for all category/kommune/institution routes

## Common Pitfalls

- `react-vendor` chunk is 220KB (slightly over 200KB target) — known, acceptable
- `chart-vendor` is 436KB — lazy loaded, only on pages with charts
- `supabase` chunk is 184KB — lazy loaded via dynamic import
- postbuild scripts need SUPABASE credentials for article pre-rendering (skips gracefully without them)
- master branch (not main) is the deploy branch
- Prerender: `prerender-pages.mjs` generates 1332 programmatic + 500 institution detail + static = 1939 total
- Sitemap: ~8500 URLs in `public/sitemap.xml` (generated by `generate-sitemap.mjs`)
- Vercel middleware: `dk-seo/middleware.ts` serves bot-specific HTML with rich structured data
- Test coverage: 6 test files, 117 tests. Core scoring (institutionScore, preferenceScore), friplads calculator, and data transforms (format, geo, dataTransform) covered.
- Analytics: PostHog + Umami in `initAnalytics.ts`. Autocapture enabled. Custom events: gate_impression, gate_email_submitted, gate_unlocked, gated_content_viewed, gate_rejection, scroll_depth, gated_feature_viewed. Per-feature tracking via `useFeatureView` hook in `src/hooks/useFeatureView.ts`.
- JSON-LD schemas in `src/lib/schema.ts` (breadcrumbSchema, itemListSchema, websiteSchema, faqSchema). All 28 pages have JSON-LD.
- 18 files exceed 400-line limit. Worst offenders were refactored: institutionScore.ts split into `src/lib/scoring/` (types, utils, school, dagtilbud, enrichment). InstitutionPage/GuidePage/HomePage had sections extracted to `src/components/detail/`, `src/components/home/`, `src/lib/guideEngine.ts`, and `src/lib/faqData.ts`.

- ESLint: remaining 53 issues are mostly `no-explicit-any` at system boundaries (Supabase responses, PostHog window casts, Recharts tooltips) and `set-state-in-effect` for legitimate async/DOM patterns. Rules-of-hooks violations were caused by useMemo after early returns — always put all hooks before conditional returns.
- `localBusinessSchema` was dead code in schema.ts — removed. Always grep for callers before assuming exports are used.
